## ⚡ Startup Pitch & Key Value Propositions

We believe finding a place to work shouldn't be a gamble. WorkSphere is a product designed to remove the guesswork of remote work by offering:

### 🧠 Cooperative Multi-Agent Intelligence
Unlike simple search boxes, our platform runs on a **5-tier cooperative AI agent mesh** (Orchestrator → Context → Data → Reasoning → Action) powered by Groq & Llama 3.3. It analyzes natural-language intentions (e.g. *"Show me a quiet cafe for a video call with strong coffee nearby"*) and details the exact thought process behind each recommended venue.

### 📡 Real-Time Telemetry & Updates
Keep-alive **Server-Sent Events (SSE)** stream live ratings and crowd density levels directly to the browser. As other users rate WiFi speeds or report socket conditions, your map instantly reflects changes without refreshing.

### 🗺️ Glowing Map & OSRM Routing
Interactive map interface built on Dark Matter styles, featuring custom user avatars and realistic **OSRM road routing**—showing the actual walking or driving path geometry instead of just straight lines.

### 📱 Offline-First PWA Core
Installable progressive web app featuring local storage fallbacks (IndexedDB) for folders, venue directories, and search history. Outbox background sync queues your likes, reviews, and updates while offline and flushes them when connectivity returns.

## 🚀 Deploying & Getting Started

### Prerequisites
* Node.js 18+
* npm or yarn
* PostgreSQL Instance (or free Neon.tech account)

### Setup & Launch
1. **Clone the repository**
   ```bash
   git clone https://github.com/SatyamPandey-07/WorkSphere.git
   cd WorkSphere
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Configure Environment**
   ```bash
   cp .env.example .env.local
   # Edit .env.local with credentials (Clerk, Database URL, Groq API key)
   ```

4. **Initialize Database**
   ```bash
   npx prisma generate
   npx prisma db push
   ```

5. **Fire up Development Server**
   ```bash
   npm run dev
   ```

## 📊 Database Schema

```prisma
model User {
  id                 String             @id
  email              String?            @unique
  createdAt          DateTime           @default(now())
  favorites          Favorite[]
  ratings            VenueRating[]
  conversations      Conversation[]
  bookings           Booking[]
}

model Venue {
  id                 String              @id @default(cuid())
  placeId            String              @unique
  name               String
  latitude           Float
  longitude          Float
  category           String              // cafe, coworking, library
  address            String?
  rating             Float?
  wifiQuality        Int?
  hasOutlets         Boolean             @default(false)
  noiseLevel         String?             // quiet, moderate, loud
  createdAt          DateTime            @default(now())
  ratings            VenueRating[]
  favorites          Favorite[]
  bookings           Booking[]
}

model Booking {
  id              String        @id @default(cuid())
  userId          String
  user            User          @relation(fields: [userId], references: [id])
  venueId         String
  venue           Venue         @relation(fields: [venueId], references: [id])
  date            String
  time            String
  status          BookingStatus @default(CONFIRMED)
  confirmationId  String        @unique
  createdAt       DateTime      @default(now())
}
```
